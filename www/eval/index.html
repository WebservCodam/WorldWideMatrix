webserv eval website

Start the server, then run each command. Every command targets 127.0.0.1:8085.

./webserv config_files/valid/eval.conf

1. GET a static file and its MIME type 200

Serves a file with the correct Content-Type based on its extension.

curl -i http://127.0.0.1:8085/images/cat.png

Returns the image with header Content-Type: image/png.

2. Autoindex (directory listing) 200

A directory with no index file and autoindex on returns an HTML listing.

curl http://127.0.0.1:8085/images/

Returns an HTML list of the files in /images.

3. Redirect 301

The /old location returns a redirect to /new.

curl -i http://127.0.0.1:8085/old

Returns 301 Moved Permanently with header Location: /new.

4. POST (upload a file) 201

The request body is written to a new file under /uploads.

curl -i -X POST --data "hello from curl" http://127.0.0.1:8085/uploads/hello.txt

Returns 201 Created. The file www/uploads/hello.txt now exists.

5. GET the uploaded file back 200

Confirms the upload landed and is served back.

curl http://127.0.0.1:8085/uploads/hello.txt

Returns 200 OK with body "hello from curl".

6. DELETE the uploaded file 204

Removes the file created by the POST above.

curl -i -X DELETE http://127.0.0.1:8085/uploads/hello.txt

Returns 204 No Content. A second DELETE returns 404.

7. Not Found (custom error page) 404

A request for a file that does not exist serves the configured error page.

curl -i http://127.0.0.1:8085/does-not-exist

Returns 404 Not Found with the custom 404 page as the body.

8. Method Not Allowed 405

/images allows only GET, so a POST is rejected and the allowed methods are listed.

curl -i -X POST --data "x" http://127.0.0.1:8085/images/cat.png

Returns 405 Method Not Allowed with header Allow: GET.

9. Payload Too Large 413

/tiny caps the body at 10 bytes, so a larger body is rejected.

curl -i -X POST --data "this body is longer than ten bytes" http://127.0.0.1:8085/tiny/x

Returns 413 Payload Too Large.

10. Virtual hosts (Host header)

Two servers share one port; the Host header decides which one answers. Run the vhosts config for this test:

./webserv config_files/valid/vhosts.conf
curl -H "Host: site_a" http://127.0.0.1:8085/
curl -H "Host: site_b" http://127.0.0.1:8085/

Returns "SITE A" for the first and "SITE B" for the second. An unknown Host falls back to the first server.

CGI: not yet implemented. Once it lands, a request to a .py script under /cgi-bin will run the script and return its output.
